## Q5. What architectural advantage does Helios gain by separating agent state checkpointing from agent control flow?

### Answer
By separating checkpointing from the main control flow, Helios keeps the **orchestrator focused on agent execution** while the **ContextGate handles checkpoint policy and briefing construction**. This modularity makes the system easier to reason about, test, and change.

The orchestrator decides *when* to checkpoint based on token thresholds, but the ContextGate decides *how* to serialize relevant state into memory and turn it into a recovery briefing. Provider implementations then separately decide *how* to reset history. This separation reduces coupling between runtime loop logic, persistence, and provider-specific behavior.

// src/core/orchestrator.ts
private async *maybeCheckpoint(session: Session): AsyncGenerator<AgentEvent> {
  const threshold = this._contextGate.checkThreshold(model, this._lastInputTokens);
  if (!threshold) return;
  const gist = await this.generateCheckpointGist(session);
  const briefing = this._contextGate.performCheckpointWithGist(gist);
  this.activeProvider.resetHistory(session, briefing);
}
