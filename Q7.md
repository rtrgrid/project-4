## Q7. Why does Helios use composable triggers instead of event-driven architectures for agent activation?

### Answer
Helios uses composable triggers because its wake-up conditions span **timers, remote process state, files, metrics, and resources**, many of which are naturally polled rather than pushed as clean application events. A pure event-driven architecture would require every external system to emit compatible events and would be harder to unify across local/remote machines.

The trigger abstraction gives Helios one consistent mechanism for agent activation, regardless of whether the underlying signal comes from time, SSH checks, log metrics, or file state. It also supports partial satisfaction and boolean composition, which is more expressive than many simple event systems.
