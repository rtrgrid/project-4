## Q3. How does Helios's metric parsing approach enable observable agent behavior without invasive instrumentation?

### Answer
Helios makes agent behavior observable by **parsing metrics directly from normal stdout/log output**, instead of forcing training code to be deeply integrated with a special instrumentation API. If a script prints values like `loss=...`, `acc=...`, or follows a custom regex pattern, Helios can extract and store those metrics automatically.

This is non-invasive because existing scripts usually already print progress to stdout. Helios only needs metric names or patterns when launching a background process, then its collector tails the log and parses new lines. That makes training behavior visible in dashboards and comparisons without requiring codebase-specific hooks.

export function patternsFromNames(names: string[]): MetricPatterns {
  patterns[name] = new RegExp(
    `(?:^|\\\\s|,)${escapeRegex(name)}\\\\s*[=:]\\\\s*([+-]?\\\\d+\\\\.?\\\\d*(?:e[+-]?\\\\d+)?)`,
    "i",
  );
}
