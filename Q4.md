## Q4. Why does Helios require explicit checkpoint boundaries rather than implicit checkpointing at arbitrary code points?

### Answer
Helios requires explicit checkpoint boundaries because it checkpoints at a **safe semantic boundary: after a provider turn completes and token usage is known**. Arbitrary checkpointing in the middle of execution could cut through streaming output, tool interactions, or partially updated state.

In the orchestrator, checkpointing happens only after `done` events and after the system has the actual context token count. Then Helios asks the model for a gist, writes durable memory, builds a briefing, and resets provider history. This keeps checkpoints coherent and reproducible.

- `src/core/orchestrator.ts`
- `src/providers/types.ts`
- `src/memory/context-gate.ts`
