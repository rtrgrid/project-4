## Q8. How does Helios's checkpoint-based recovery enable transparent agent migration across hosts?

### Answer
Helios's recovery model depends on **serialized durable state** rather than a live in-memory process image. Because the recovery artifact is a memory-tree briefing plus persisted session state, the resumed agent only needs access to that stored state and the ability to continue from the briefing.

That means recovery is host-independent in principle: instead of resuming a frozen process image tied to one machine, Helios reconstructs context from storage and injects it back into the provider history via `resetHistory`. This is what makes host migration transparent at the architectural level.
